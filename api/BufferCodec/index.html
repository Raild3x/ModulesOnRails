BufferCodec

Compact binary value codec over Luau buffers — the byte-level layer of the replication wire (Wire.luau is the message layer on top). Every value starts with a one-byte tag, and the common cases bit-pack their payload into the tag byte itself so they cost zero extra bytes: integers 0..127 and -32..-1, string lengths 0..31, array/map sizes 0..15. Larger values fall through to explicit-width tags, with numbers stored in the smallest integer width that holds them (or f32 when it round-trips losslessly, f64 otherwise).

Strings of length >= 2 are interned per message: the first occurrence writes the bytes, repeats write a 2-3 byte back-reference. Repeated dictionary keys (arrays of uniform records in a snapshot) collapse to almost nothing. Reader and writer must apply the same interning rule (INTERN_MIN_LENGTH / INTERN_MAX_COUNT) or references desync.

Instances cannot live inside a buffer — only the engine can serialize an instance ref, and only as a remote argument. The writer therefore collects them into a side list (deduplicated) and stores a 1-based index in the buffer; export returns the list so the message layer can ship it as a second remote argument, and the reader resolves indices against the received list. A ref the engine can not resolve on the client (streamed out, server-only) decodes as nil.

Any value the buffer has no tag for (Roblox datatypes without a dedicated tag — Ray, Region3, UDim2, EnumItem, … — as well as anything else) rides that same side list rather than being rejected: the codec hands the raw value to the remote to serialize natively, exactly like an Instance ref. The buffer stores only the index. This keeps unsupported data flowing (at native remote cost) instead of erroring out the whole message.

A value a RemoteEvent genuinely can't serialize (function, thread, RBXScriptSignal, RBXScriptConnection — see NON_REPLICABLE_TYPES) can't ride the side list either: sending it would throw at FireClient and take the whole batched message down. The writer instead warns and encodes nil in its place, so that one bad value is dropped in isolation while every unrelated op sharing the frame still delivers.

Directly-tagged value types (bit-packed into the buffer): nil, boolean, number, string, table, Vector3, Vector2, CFrame, Color3. Instances and everything else ride the side list.

newWriter​

</>
BufferCodec.newWriter() → Writer

Creates a new writer used for encoding values into a binary buffer.

The writer maintains temporary state while encoding, including string interning tables and deduplicated instance references. 

local writer = BufferCodec.newWriter()

BufferCodec.writeValue(writer, {
	Name = "Player",
	Health = 100,
})

local buffer, instances = BufferCodec.export(writer)

buffer,
{any}?

Finalizes a writer.

Returns a tightly-sized buffer containing the encoded bytes and an optional side list of raw values (Instances and any untagged types) which must accompany the buffer when transmitting over a RemoteEvent. 

local buffer, instances = BufferCodec.export(writer)

RemoteEvent:FireClient(player, buffer, instances)

newReader​

</>
BufferCodec.newReader(
buffer: buffer,
instances: {any}?
) → Reader

Creates a reader for decoding values from an encoded buffer.

The supplied side list should be the same list returned by export on the writing side. 

local reader = BufferCodec.newReader(buffer, instances)

local value = BufferCodec.readValue(reader)
